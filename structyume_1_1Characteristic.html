Detailed Description

A criterion for classifying a stream of characters as a specific type of token Token::Type.

TODO: requires doc update regarding TokenState (and TokenState itself needs docs)

The criterion is a function taking three arguments: an int representing the current character being evaluated, a second int representing the index of this character relative to the current character sequence (that is, since the end of the previous token), and a stringstream, where matching characters should be written to. The function should return a bool, whether or not this character is viable as this type of token. As soon as false is returned, this token ends and the process is repeated.

Note that while usually the same character (the first int argument) should be appended, tokens such as string literals may include escapes and thus output the corresponding unescaped value.
